# CrewOps M6 Demo Walkthrough

This walkthrough is the shortest operator demo that exercises the M6 contracted-service surface without leaving the deterministic seed.

## Recommended Flow

1. Open `estimates`.
   Use `EST-6002` to show a sent estimate that still needs customer approval.
2. Record approval.
   Enter the signature name and approval note, then run `Approve`.
3. Convert the estimate.
   Run `Convert` to show the estimate-to-contract handoff and linked agreement selection.
4. Open `contracts`.
   Show the selected agreement, then create a new draft contract from the contract action card.
5. Activate the new agreement.
   Run `Resume` on `agreement_004` to show the post-create activation path and recurring-plan activation.
6. Open `recurring`.
   Select `PLAN-001` to show future schedule generation, then `PLAN-002` for skip-and-resume recovery.
7. Open `integrations`.
   Load deliveries, then run `Retry Failed` to show the retry path and the `delivery_retry_status` plus `stale_delivery_id` sync feedback.
8. Close on `manager` or `finance`.
   Use contract health, renewal, recurring revenue, receivables, and profitability cards to show that the M6 commercial layer stays integrated with the M5 finance layer.

- `EST-6002` is the canonical sent estimate for approval and conversion replay.
- `agreement_004` is the canonical post-create activation case.
- `delivery_002` is the canonical failed webhook delivery used for retry replay.
- The demo seed is generated by `scripts/ci/seed_demo.sh`; if any ids drift, regenerate the seed before updating docs or traces.
